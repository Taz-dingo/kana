# Kana Plan

## 当前推进策略

`kana` 现在应该先从“正确的基础学习产品”做起，而不是急着扩张功能面。

推进顺序：

1. 把基础文档和产品约束写清楚
2. 修正数据结构与五十音表准确性
3. 持续打磨总览与详情体验
4. 补齐发音资源
5. 加入最小可用的练习模式
6. 建立最小可用的记忆系统

## 阶段拆分

### Phase 0：文档化与对齐

目标：把产品目标、设计方向、当前问题、执行任务沉淀下来。

输出：

- 文档骨架建立完成
- 参考案例被拆解为可执行原则
- 当前代码问题被明确记录

### Phase 1：数据模型与总览校正

目标：让五十音数据表达方式正确，并让总览界面不再依赖错误的重复占位。

重点：

- 支持空位而不是重复字符
- 明确行列语义
- 为后续浊音 / 拗音扩展留出结构空间
- 校正字源字段和详情数据来源

### Phase 2：详情体验与发音

目标：把当前详情弹窗继续打磨成更完整的学习详情层。

重点：

- 更清晰地展示平 / 片对应关系
- 强化字源信息的阅读体验
- 接入可用的发音播放
- 改善移动端与键盘交互

### Phase 3：基础练习模式

目标：从“可看”升级到“可学”。

重点：

- 看字选音
- 看音选字
- 听音识字
- 记录简单的正确率与错题

### Phase 4：记忆系统 MVP

目标：把五十音从“可练习内容”升级为“可调度学习项”。

重点：

- 建立 `KanaMemoryState`
- 建立本地记忆状态存储
- 区分“新内容”和“待复习内容”
- 增加最小可用的复习调度规则
- 建立 Again / Hard / Good / Easy 或等价反馈

### Phase 5：扩展与长期学习系统

目标：逐步扩展为更完整的假名学习入口。

重点：

- 浊音 / 半浊音 / 拗音
- 学习进度
- 错题回顾
- 更接近 FSRS-lite 的复习节奏设计

## 关键决策

### 决策 1：文档优先

从现在开始，`kana` 的产品方向和设计取舍优先记录到 `docs/`，避免边做边漂移。

### 决策 2：以学习闭环为核心

页面结构和技术实现都围绕“总览 → 详情 → 发音 → 练习 → 复习”的闭环推进。

### 决策 3：参考外部产品，但不照搬旧 UI

参考站点在信息组织上有价值，但视觉实现偏旧。`kana` 应吸收其学习结构，使用更现代的界面表达。

### 决策 4：记忆系统先做简化版，再考虑高阶建模

现阶段不追求一开始完整实现 `FSRS`。

优先做：

- 可解释
- 可调试
- 可迭代
- 对初学者负担友好

的 `FSRS-lite`。

## 待确认项

- 发音资源最终如何分批补齐本地音频
- 详情弹窗中的字源与记忆提示模块如何继续细化
- 首批练习模式只做一种，还是同步做两种
- 记忆系统第一版的评分按钮文案最终采用中文描述还是保留 Anki 式术语
- 记忆状态是否先做本地存储，还是从一开始就考虑可同步结构
