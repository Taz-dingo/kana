# Kana Spec

## 项目定位

`kana` 是一个面向日语初学者的五十音学习产品。

它的核心任务不是展示“日语字母表”，而是帮助用户建立以下几层认知：

1. 这个字符读什么
2. 这个字符对应的平假名 / 片假名是什么
3. 这个字符从哪里来，为什么长这样
4. 这个字符如何发音、如何被反复记住
5. 学完之后，能不能快速自测并形成记忆闭环

所以，`kana` 不应该停留在静态信息页，而要逐步成为一个“预览 + 理解 + 发音 + 练习”的学习界面。

## 设计哲学

### 1. 先帮助记住，再考虑堆功能

每个页面和交互都应该服务于“更容易记住五十音”。

如果一个功能不能帮助用户更快辨认、理解、回忆或练习，就不应该优先进入第一阶段。

### 2. 从总览进入细节

五十音学习天然有两种视图：

- 总览：快速扫一遍整个音节系统
- 单字：深入理解某个假名的读音、字形、字源、笔顺

产品应该允许用户先看全局，再通过点击进入单字深读，而不是一上来把信息塞满。

### 3. 学习型界面，不是资料堆叠页

信息要足够完整，但不能杂乱。界面要为学习服务：

- 让字符本身成为第一视觉重心
- 让读音、对应假名、字源成为清晰的二级信息
- 把笔顺、发音、练习等交互收纳为明确动作，而不是平铺噪音

### 4. 视觉上克制，但情绪上要温和有吸引力

这个项目更适合“安静、清晰、专注”的气质，而不是花哨或游戏化过度的表达。

视觉基调应偏向：

- 干净留白
- 强字符排版
- 少量强调色
- 明确层级
- 移动端优先但桌面端同样舒适

### 5. 准确性优先于速度

五十音表的数据结构、空位处理、字源描述、发音映射都必须准确。

如果基础数据不准确，再漂亮的 UI 也只是在放大错误。

## 产品目标

## 短期目标

- 做出准确、可点击的五十音总览
- 在单字详情中展示字源信息
- 提供稳定、易用的发音播放
- 明确空位与特殊行，不用错误重复值硬填表格

## 中期目标

- 加入基础练习模式
- 支持平假名 / 片假名联动记忆
- 形成学习闭环：预览 → 详情 → 听音 → 自测

## 长期目标

- 扩展到浊音、拗音、长音、促音等更完整的假名系统
- 引入记忆进度、错题回顾、个性化复习
- 让 `kana` 成为日语学习项目中的基础入口，而不是一次性页面

## 用户体验原则

### 首屏原则

- 首屏应该直接进入五十音学习本体，而不是先被说明文字挡住
- 用户进入页面后，应在几秒内理解“这是可以点开的五十音总览”

### 信息层级原则

- 一级信息：假名字符本身
- 二级信息：罗马音、对应片 / 平假名
- 三级信息：字源、笔顺、发音控制、补充说明

### 交互原则

- 单击字符即可进入详情
- 发音按钮必须一眼可见，但不喧宾夺主
- 弹层或详情面板应支持键盘关闭、遮罩关闭、明确的焦点管理

### 学习原则

- 每个假名至少要支持“看字识音”和“听音识字”两种学习路径
- 字源不是装饰性文案，而是帮助记忆的助记信息
- 发音是核心能力，不是附加彩蛋

## 范围边界

## 第一阶段必须包含

- 五十音总览
- 平假名 / 片假名切换或联动展示
- 单字详情
- 字源展示
- 发音播放
- 正确的空位处理

## 第一阶段暂不强求

- 登录系统
- 云同步
- 排行榜
- 重度游戏化机制
- 过多自定义主题或装饰性动效

## 非目标

当前阶段不把 `kana` 做成完整日语课程平台，也不急着覆盖语法、词汇、例句体系。

先把“五十音学习体验”做到扎实、舒服、可持续迭代。

## 视觉方向

参考页说明了一种值得借鉴的学习结构：

- 先看总览卡片
- 点击进入单字详情
- 详情里出现字源、笔顺、发音

但 `kana` 不需要照搬它的视觉风格。我们的方向是：

- 借鉴它的信息组织和学习闭环
- 用更现代、更克制、更干净的界面语言重做
- 尽量减少旧式卡片堆叠感，提高字符与内容层级的可读性

## 发音策略方向

发音能力需要从第一阶段开始纳入设计，而不是后补。

初步策略：

- 单字详情提供显式播放按钮
- 总览视图可考虑为每个单元格提供轻量播放入口或长按播放
- 优先保证每个假名都有可预测、稳定的发音体验

待实现阶段再决定具体技术方案：

- 方案 A：内置本地音频资源
- 方案 B：浏览器语音能力作为兜底
- 方案 C：本地资源 + 浏览器兜底混合方案

默认倾向混合方案，因为它更利于可控质量与低接入门槛。
