# Kana Status

## 当前阶段

项目目前处于“基础学习体验已成型，正在向练习与记忆系统过渡”的阶段。

已具备可浏览、可点开详情、可发音、可连续浏览的基础学习链路，并已完成一轮关于 Anki / FSRS / 记忆研究的方向调研。

## 已有能力

- 已有 Astro + React + Tailwind 的基础项目结构
- 已有单页面入口
- 已支持平假名 / 片假名切换
- 已支持点击假名查看详情
- 已在详情弹窗中展示罗马音、平/片对应、部分字源与备注
- 已修正 `や行`、`わ行` 的空位表达，不再使用重复字符占位
- 已在详情层中同时展示平 / 片假名对应关系
- 已加入“本地音频优先、浏览器 TTS 兜底”的首版发音结构
- 已支持详情层连续浏览、上一个 / 下一个切换与键盘方向键切换
- 已修正中 / 日文字形策略，确保中文 UI 与假名字形分别走合适字体栈
- 当前构建可通过

## 已沉淀的新研究

- 已明确 `kana` 后续的记忆系统应受 `Anki / FSRS` 启发，但不需要第一版就完整复刻
- 已明确真正有强证据支持的是“间隔练习 + 主动回忆”，而不是简单宣传“遗忘曲线”
- 已明确后续产品叙事应避免神化“固定遗忘曲线”，转而强调可持续复习与记忆调度
- 已形成两份专项文档：
  - `docs/memory-research.md`
  - `docs/memory-system.md`

## 当前缺口

### 数据层

- 基础数据结构已支持空位，但还未拆出独立的记忆状态层
- 字源字段当前以平假名为主，片假名来源仍需继续补充
- 当前模型仍需继续为浊音、拗音与特殊音节扩展做准备

### 交互层

- 详情弹窗已经可用，且支持连续浏览，但“练习态”和“复习态”界面尚不存在
- 发音能力已预留本地音频字段，但仍未真正补齐音频资源
- 没有正式练习模式
- 没有记忆状态、复习列表或今日任务

### 视觉层

- 页面风格已调整为更温和、更适合语言学习的浅色界面
- 总览 / 详情节奏已更清楚，并补充了连续浏览反馈
- 当前视觉语言已经收敛，但后续练习与复习面板还没有视觉规范

### 可访问性

- 详情层已补上 Esc、Tab 限制与关闭后回焦，仍可继续完善桌面端细节
- 发音状态反馈已具备基础状态展示，但仍可继续加强
- 练习模式的键盘操作与评分反馈还未开始设计

## 当前判断

项目已经从“静态五十音页面”转向“基础学习产品”，现在最应该做的不是继续堆展示信息，而是：

1. 建立最小练习模式
2. 建立记忆状态层
3. 把学习流变成“新学 + 复习”的闭环

## 下一阶段建议

最适合紧接着推进的方向是：

- 先实现 `KanaMemoryState` 与本地存储
- 先做一版 `romaji -> kana` 主练习
- 先做简化的 `Again / Hard / Good / Easy` 调度
- 再决定是否扩展到更多题型与更接近 `FSRS` 的建模
