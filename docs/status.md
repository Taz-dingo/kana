# Kana Status

## 当前阶段

项目目前处于“基础学习体验已成型，已进入练习与记忆系统 MVP”的阶段。

已具备可浏览、可点开详情、可发音、可连续浏览的基础学习链路，并已完成一轮关于 Anki / FSRS / 记忆研究的方向调研，同时已经接入首版本地记忆状态、学习面板、最小练习闭环与练习总结反馈。

## 已有能力

- 已有 Astro + React + Tailwind 的基础项目结构
- 已有单页面入口
- 已支持平假名 / 片假名切换
- 已支持点击假名查看详情
- 已在详情弹窗中展示罗马音、平/片对应、部分字源与备注
- 已修正 `や行`、`わ行` 的空位表达，不再使用重复字符占位
- 已在详情层中同时展示平 / 片假名对应关系
- 已加入“本地音频优先、浏览器 TTS 兜底”的首版发音结构
- 已支持详情层连续浏览、上一个 / 下一个切换与键盘方向键切换
- 已修正中 / 日文字形策略，确保中文 UI 与假名字形分别走合适字体栈
- 已修复详情弹窗打开时的硬切换问题，当前打开 / 关闭过渡已基本平滑
- 已建立 `KanaMemoryState` 类型与本地存储 repository
- 已建立首版 scheduler，可计算 `due`、`new`、`learning`、`mastered` 等状态统计
- 已在总览右侧接入“今日待复习 / 待学习新字符”面板
- 已加入学习面板，可区分“复习优先”和“新学训练”
- 已实现两种首版题型：`romaji -> kana` 与 `kana -> romaji`
- 已实现首版评分回写：`again / hard / good / easy`
- 已增加练习结束后的总结页与错题摘要
- 已支持“只练错题”的 session 内再训练
- 已加入跨 session 的弱项强化入口，可集中练习最近最不稳的字符
- 当前构建可通过

## 已沉淀的新研究

- 已明确 `kana` 后续的记忆系统应受 `Anki / FSRS` 启发，但不需要第一版就完整复刻
- 已明确真正有强证据支持的是“间隔练习 + 主动回忆”，而不是简单宣传“遗忘曲线”
- 已明确后续产品叙事应避免神化“固定遗忘曲线”，转而强调可持续复习与记忆调度
- 已形成两份专项文档：
  - `docs/memory-research.md`
  - `docs/memory-system.md`

## 当前缺口

### 数据层

- 记忆状态层已存在，但还没有用户级参数、弱项聚类或远端同步结构
- 字源字段当前以平假名为主，片假名来源仍需继续补充
- 当前模型仍需继续为浊音、拗音与特殊音节扩展做准备

### 交互层

- 已有正式练习模式和总结页，但长期错题队列与跨 session 弱项强化尚未建立
- “今日任务完成”反馈和连续学习节奏还未形成
- 发音能力已预留本地音频字段，但仍未真正补齐音频资源

### 视觉层

- 页面风格已调整为更温和、更适合语言学习的浅色界面
- 总览 / 详情 / 学习面板的节奏已更清楚
- 练习态已经形成，但结果页、复习总结页和弱项面板还可以继续统一和增强

### 可访问性

- 详情层已补上 Esc、Tab 限制与关闭后回焦，仍可继续完善桌面端细节
- 发音状态反馈已具备基础状态展示，但仍可继续加强
- 练习模式还没有完整的键盘快捷答题与结果提示结构

## 当前判断

项目已经从“静态五十音页面”转向“具备最小学习闭环的基础学习产品”，现在最应该做的是：

1. 增加“今日任务完成”与复习节奏反馈
2. 继续增强跨 session 弱项解释与弱项回看
3. 继续把学习流变成“新学 + 复习 + 再练”的闭环

## 下一阶段建议

最适合紧接着推进的方向是：

- 增加“今日任务完成”与 session 结束反馈
- 继续增强跨 session 的错题回看与弱项解释
- 再决定是否扩展到更多题型与更接近 `FSRS` 的建模
