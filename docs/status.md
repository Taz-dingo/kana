# Kana Status

## 当前阶段

项目目前处于“基础学习体验已成型”阶段。

已具备可浏览、可点开详情、可发音的基础学习链路，但离完整课程化产品还有明显差距。

## 已有能力

- 已有 Astro + React + Tailwind 的基础项目结构
- 已有单页面入口
- 已支持平假名 / 片假名切换
- 已支持点击假名查看详情
- 已在详情弹窗中展示罗马音、平/片对应、部分字源与备注
- 已修正 `や行`、`わ行` 的空位表达，不再使用重复字符占位
- 已在详情层中同时展示平 / 片假名对应关系
- 已加入“本地音频优先、浏览器 TTS 兜底”的首版发音结构
- 当前构建可通过

## 当前缺口

### 数据层

- 基础数据结构已支持空位，但还未拆出更系统的扩展层
- 字源字段当前以平假名为主，片假名来源仍需继续补充
- 当前模型仍需继续为浊音、拗音与特殊音节扩展做准备

### 交互层

- 详情弹窗已经可用，但信息层次和字源模块仍可继续加强
- 发音能力已预留本地音频字段，但仍未真正补齐音频资源
- 没有练习模式
- 没有进度或反馈机制

### 视觉层

- 页面风格已调整为更温和、更适合语言学习的浅色界面
- 总览 / 详情节奏已更清楚，但仍可继续打磨内容密度与视觉完成度
- 当前视觉语言已经收敛，但还可以继续补充更细的教材感细节

### 可访问性

- 详情层已补上 Esc、Tab 限制与关闭后回焦，仍可继续完善桌面端细节
- 发音状态反馈已具备基础状态展示，但仍可继续加强

## 最近确认的新方向

- 参考页的“总览 + 点击详情 + 字源 + 发音”结构很接近目标
- `kana` 后续会明确加入发音能力
- 从现在开始把文档沉淀视为正式工作流的一部分

## 当前判断

这个项目适合继续做，而且值得做。

原因不是它已经完成很多，而是它的目标边界足够清晰：

- 场景明确
- 内容聚焦
- 交互闭环天然成立
- 小而完整，适合逐步打磨

目前最应该做的不是继续堆小功能，而是先把基础模型、参考方向和实现顺序稳定下来。
