# Kana References

## 参考页

- [五十音在线学习](https://nya.ink/50yin/index.html)

## 这页为什么值得参考

这不是因为它的视觉设计多先进，而是因为它已经把“五十音学习”里几个关键动作连起来了：

- 总览浏览
- 点击单字进入详情
- 查看字源
- 查看笔顺
- 播放发音
- 后续还衔接测验

也就是说，它展示的是一条完整的学习路径，而不是孤立页面。

## 从参考页提炼出的有效点

### 1. 总览和单字详情是天然的两层结构

用户先看全表，再点进单字深读，这是很自然的学习路径。

### 2. 字源信息有记忆价值

“字源”不是附属冷知识，而是帮助初学者把抽象假名和具体汉字形态联系起来的重要助记点。

### 3. 发音按钮应直接可用

发音不是隐藏功能。用户在浏览和详情中都可能随时想听。

### 4. 学习产品需要闭环

如果只有展示，没有测验和反馈，记忆很难巩固。

## 不建议直接照搬的点

- 旧式 Material 卡片堆叠感较重
- 视觉层级偏碎，信息密度高但不够克制
- 组件和按钮风格年代感明显
- 首屏气质更像工具页，而不是精心打磨的学习产品

## 更高层的语言学习产品启发

在扩大到语言学习产品后，可以提炼出更稳定的共性：

- 好产品会明确区分“学新内容”和“复习旧内容”
- 好产品不会只做展示，而会安排短时、高频、可完成的练习闭环
- 好产品的进度系统不只显示完成率，还会表达当前状态、下一步和薄弱点
- 好产品会逐步去掉提示，而不是一直依赖辅助信息

## 记忆系统层的参考价值

在记忆系统层面，`Anki / FSRS` 提供的启发不在视觉，而在机制：

- 把内容当作可调度学习项
- 把复习时机看作系统能力
- 把“主动回忆”而不是“再次阅读”视为核心动作
- 在保留率与复习负担之间做平衡

这部分研究已单独沉淀到：

- `docs/memory-research.md`
- `docs/memory-system.md`

## 对 `kana` 的实际启发

`kana` 应该同时吸收两类参考：

- 学习结构参考：总览、详情、发音、练习、反馈
- 记忆系统参考：调度、回忆、复习、进度、弱项强化

更具体地说：

- 学展示型产品的信息组织
- 学语言学习产品的学习闭环
- 学 `Anki` 类系统的记忆调度思想
- 不照搬任何单一产品的旧 UI 或过重机制
