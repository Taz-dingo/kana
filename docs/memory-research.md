# Kana Memory Research

## 目的

这份文档用来沉淀 `kana` 项目关于记忆系统、间隔重复、Anki / FSRS 与遗忘研究的结论。

它不追求做成完整文献综述，而是回答三个对产品最关键的问题：

1. `Anki` 这套记忆系统为什么有效
2. “遗忘曲线”这类说法哪些成立，哪些被过度简化
3. 对 `kana` 这种小粒度语言学习产品，哪些机制值得借

## 研究结论总览

### 结论 1：`Anki` 有效，但有效的关键不是“神秘算法”

`Anki` 的有效性主要来自几个已经被大量研究支持的学习原则：

- 间隔练习（spacing / distributed practice）
- 主动回忆（retrieval practice / testing effect）
- 及时反馈
- 根据学习表现动态安排复习时间

所以，`Anki` 的价值不在于“它精确复刻了某条固定遗忘曲线”，而在于它把这些原则变成了长期可执行的复习系统。

### 结论 2：“遗忘曲线”可以当启发，但不能当教条

传统表述里常把记忆遗忘理解为一条单一、固定、对所有人都适用的曲线。

这类表达有启发性，但太粗糙。

更接近当前研究的说法是：

- 记忆确实会随时间下降
- 复习确实会改变后续遗忘速度
- 但不同材料、不同任务、不同时间段、不同个体，遗忘形态并不完全相同
- “单一标准曲线”不足以描述真实学习过程

### 结论 3：对语言学习产品来说，`检索 + 调度` 比“展示内容”更重要

真正能拉开体验差距的，不是把知识卡片做得更花，而是：

- 用户是否真的在回忆
- 系统是否真的知道什么该复习
- 复习是否控制在可持续负担之内

## Anki / FSRS 的要点

### `Anki` 现在的方向

`Anki` 官方当前已经把 `FSRS` 作为关键调度方案之一。

从官方文档与 FAQ 来看，`FSRS` 的重要特点是：

- 允许用户设定目标记忆率（`desired retention`）
- 根据历史复习数据优化参数
- 在“尽量少复习”和“尽量少遗忘”之间做权衡
- 比旧式固定启发式调度更个性化

### `desired retention` 的产品意义

官方默认推荐值是 `0.90`。

这件事很重要，因为它直接揭示了一个产品事实：

> 记忆系统不是追求“尽可能不忘”，而是在“保留率”和“工作量”之间做现实平衡。

对 `kana` 来说，这意味着：

- 不应该默认追求极高记忆率
- 复习系统必须考虑用户每天能承受多少负担
- “更稳”通常意味着“更多复习”

### `FSRS` 的建模思路

`FSRS` 社区公开资料把系统的核心状态理解为：

- difficulty：这项内容对用户有多难
- stability：这项内容在当前状态下能稳定多久
- retrievability：此刻还能回忆出来的概率有多高

这比“固定乘 2、乘 3 的间隔倍率”更像一个动态记忆模型。

对 `kana` 的启发是：

- 未来不应该只记录“答对 / 答错”
- 更应该记录“这个字符现在有多稳”
- 系统要依据稳定度决定下次什么时候回来

## 科学证据梳理

### 1. 间隔练习证据很强

经典综述与后续研究都普遍支持：

- 分散学习通常优于临时突击
- 适度拉开复习间隔，能提升长期保持

这说明 `kana` 后续不应该只提供自由浏览，而应该提供“延迟后再提取”的机制。

### 2. 检索练习证据很强

大量研究支持：

- 单纯重看材料不如先尝试回忆再核对答案
- 主动提取本身会强化记忆痕迹
- 长期保持尤其依赖反复提取，而不是反复暴露

所以对 `kana` 来说：

- “看字符详情”只能算输入
- 真正的记忆系统必须包含回忆型题目
- 练习模式不能只做展示与点击翻面

### 3. 间隔与检索结合，比单独任一机制更有价值

Anki 的实际优势就在这里：

- 用户先尝试回忆
- 系统根据表现和时间安排下一次出现

这比“每天刷一遍全表”要有效得多，也更可持续。

## 常见误解

### 误解 1：只要按遗忘曲线安排时间就一定高效

不准确。

如果内容设计不好、提示过强、用户其实没有认真回忆，那么再好的调度也会失效。

### 误解 2：记忆系统等于语言学习系统

不准确。

记忆系统擅长：

- 假名
- 词汇
- 术语
- 小型规则

但它不能单独替代：

- 语法理解
- 阅读能力
- 对话能力
- 开放表达能力

所以 `kana` 可以把记忆系统做得很强，但它依然只是语言学习产品中的一个基础层。

### 误解 3：复习越多越好

不准确。

复习负担如果过大，用户会放弃。

一个好的记忆系统必须同时优化：

- 留存率
- 时间成本
- 心理负担
- 每日可持续性

## 对 `kana` 的设计启发

### 1. `kana` 非常适合记忆系统

因为五十音天然具备这些特征：

- 粒度小
- 可独立建模
- 可快速判断对错
- 容易混淆
- 需要长期保持

这正是间隔重复系统最擅长的内容类型。

### 2. 第一个版本不用直接复刻 `FSRS`

`kana` 当前阶段更适合先做一个 `FSRS-lite`：

- 先建立基本复习状态
- 先用简化的稳定度与调度规则
- 后续再考虑更精细的参数化建模

### 3. 练习设计必须围绕“检索”而不是“展示”

后续题型应该优先做：

- `kana -> romaji`
- `romaji -> kana`
- `audio -> kana`
- 易混淆对比

而不是继续堆更多“说明页”。

### 4. 进度系统不应该只显示完成率

更关键的是要表达：

- 哪些字符未学
- 哪些字符正在学习中
- 哪些字符不稳
- 哪些字符需要今天复习
- 哪些字符已经稳定

## 对产品叙事的实际影响

如果后续要把 `kana` 介绍成一个更完整的学习产品，我们更应该这样描述它：

- 它使用经过研究支持的间隔练习与主动回忆原则
- 它帮助用户在合适的时间复习最容易忘的字符
- 它不是简单背表，而是一个逐步建立长期记忆的训练系统

而不应该过度宣传：

- “完全基于艾宾浩斯遗忘曲线”
- “算法保证不会忘”
- “几天就能永久记住”
